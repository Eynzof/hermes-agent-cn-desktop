## 0. 跨页共通规格

不每页重复，统一定义：

### 0.1 加载态层级

| 层 | 用法 | 示意 |
|---|---|---|
| **页面级 skeleton** | 路由切换 + 数据未就绪 | 主区显灰色块占位，保留侧栏 |
| **区块级 spinner** | 局部数据刷新 | 单个卡片右上角小转圈 |
| **inline 字段 spinner** | 字段验证 / 测试中 | 输入框右侧小转圈 |
| **乐观更新** | 启用/禁用、星标等可逆操作 | 立即更新 UI，失败回滚 + toast |

**禁止**：整页大转圈遮罩。永远显示部分内容（侧栏 / 顶栏）。

### 0.2 错误态分级

| 级 | 场景 | UI |
|---|---|---|
| **致命**（无法继续） | Dashboard 不可达 / Token 全过期 / 后端协议不兼容 | 全屏错误页 + 「重连」「打开设置」按钮 |
| **页面错误** | 列表 API 失败 | 主区显错误卡 + 「重试」按钮，侧栏保留 |
| **操作错误** | 单次按钮点击失败 | toast（5s 自消失，可手动关）+ 控制台留 log_id |
| **校验错误** | 表单字段格式不对 | inline 红色提示 + 阻止提交 |

所有错误**必须**带 `log_id`（飞书 / Hermes Dashboard 都会返回），方便日志反查。

### 0.3 空状态

每页空状态都要有：

- 1 个图标（lucide-icons，弱化色调）
- 1 句话说明（不是"暂无数据"，是"还没有 X，可以做 Y"）
- 1 个主行动按钮（如果适用）

例："还没有任务，按 `⌘ N` 起一个" + 按钮「新建任务」。

### 0.4 破坏性操作确认

删除 / 禁用 / 切 profile 等：

- 二级确认 modal，文案明确「这会 ...，操作不可恢复」
- 「取消」是默认 focus（按 Enter 默认取消，避免误删）
- 「确认」红色按钮，文案不是「确认」而是「删除 X」

### 0.5 SSE 流式内容

任务详情 / 日志 / 终端 用 SSE。统一行为：

- 连接断了**不**清空已有内容
- 顶部出现 「连接断开，正在重连…」横条
- 重连成功后横条变成 「已恢复」 2 秒后自动消失
- 用户主动断开（关页面 / 切路由）走 EventSource.close()
- 重连失败 5 次后停止，显示 「无法连接，[手动重试]」

## 1. 工作台分组（01 顶导）

### 1.1 `/` 工作台主仪表盘（PanelRoute, 原型 01）

**用途**：进入 app 后默认页。一屏看见健康 + 进行中 + 写任务入口。

**布局**：

```
[顶栏（全局）]
[侧栏: 项目快捷 + 进行中会话 + 新任务] [主区]

主区从上到下：
  ├── Hero 问候（含当前 profile / workspace）
  ├── Composer 嵌入版（点击展开 / ⌘N 跳 /new）
  ├── 健康检查 7 网格（gateway / token / home / model / skills / mcp / workspace）
  ├── 进行中任务卡（最多显示 3 个，超过显示「查看全部」）
  ├── 配方 / 模板卡（最多 6 张）
  └── 跨项目活动 Feed（最多 10 条，超过显示「查看全部」跳 /history）
```

**主要数据**：
- 7 项健康探针（轮询 30s 或 SSE 推送）
- 进行中任务列表（轮询 5s）
- 最近会话 Feed（拉一次，SSE 增量更新）

**空状态分支**：

| 场景 | 渲染 |
|------|------|
| 首次启动（无任务、无历史） | 01b 空状态版 — 入门 3 步 + Preflight 清单 |
| 有 profile 但无任务 | 健康网格正常 + Composer + 「还没起任务，写一个吧」 |
| 全部健康项 healthy | 健康网格收起为 1 行徽标 |
| 有健康项异常 | 异常卡片展开，hover 显示 hint，点击跳对应修复页 |

**错误态**：

| API | 失败处理 |
|----|---------|
| 健康探针失败 | 对应那项显灰色 + 「检测失败」+ 重试按钮，不影响其它项 |
| 任务列表失败 | 任务区块显错误卡 + 重试，其它区块正常 |
| 活动 Feed 失败 | Feed 区块显错误卡，其它区块正常 |

**关键交互**：

- 点 Composer → 展开成完整编辑器（不跳路由，原地展开）
- 点「展开为新任务」→ 跳 `/new`，已填内容带过去
- 点进行中任务卡 → `/tasks/:id`
- 点活动 Feed 某行 → `/tasks/:id`
- 点配方 → 跳 `/new` + 预填模板内容
- ⌘N → 跳 `/new`

**边界**：

- 进行中任务 > 3 时，第 3 张卡变成「+ N 个进行中」点击跳 `/history?status=running`
- 健康网格中某项的 hint 跳页是 `/settings?section=models` 这种 deep link，不开 modal
- 横屏窄（< 1280px）：配方卡换成 2 列；< 1024px：单列堆叠

### 1.2 `/new` 新建任务（NewTaskRoute, 原型 02 / 02b）

**用途**：写完整任务提示词 + 选工作区 / 模型 / 审批策略 + 预检查。

**布局**：

```
[侧栏（项目快捷 + 进行中会话）] [主区]

主区从上到下：
  ├── Composer
  │     ├── 提示词编辑框（多行）
  │     ├── 附件列表（拖拽 / @file 添加）
  │     ├── 底部选择行：工作区 / 模型 / 审批策略 / Skills 标签
  │     └── 字符计数 + Token 估算 + ⌘↵ 发送
  ├── Preflight 清单（8 项）
  └── 元数据卡（工作区 / 模型 / 消费预估）
```

**字段定义**：

| 字段 | 必填 | 默认值 | 校验 |
|------|------|------|------|
| 提示词 | ✓ | 空 | 非空；长度 < 模型上下文限 80% 时绿色，> 80% 黄色，> 100% 红色阻止发送 |
| 工作区 | ✓ | 上次用的（UI SQLite） | 必须存在 + 可访问，否则红色 |
| 模型 | ✓ | profile 默认 | 必须 healthy + token 未过期 |
| 审批策略 | ✓ | profile 默认 | 三选一：每步审批 / 仅高风险 / 自动批 |
| 附件 | — | 空 | 单文件 < 100MB；总和 < 500MB（v2 限制） |
| Skills 标签 | — | 空 | 每个 Skill 必须 enabled |

**Preflight 8 项**（PRD § 5.1）：

1. 工作区路径存在且可读写
2. 模型 healthy 且 token 未过期
3. 所选 Skills 全部 enabled
4. MCP 服务器（若 Skill 依赖）全部在线
5. 当前 profile 配额未超
6. 工作区无 git lock / 大量未提交（warn 级，不阻塞）
7. 附件大小总和未超限
8. 提示词长度未超模型上下文

每项状态：⏳ 检查中 / ✅ 通过 / ⚠️ 警告 / ❌ 失败。

**主要行动**：

- ⌘↵ 或点「发送」：所有 ❌ 检查必须先解决；⚠️ 可跳过；发送后跳 `/tasks/:newTaskId`
- 「保存为草稿」：保存到 UI SQLite（v2 不入后端）
- 「应用配方」：从 Hero 区配方卡过来时已预填，可再选其它

**空状态**：

- 直接进 `/new` 不带预填 → Composer 显占位文本「描述你要让 Hermes 做的事…」
- Preflight 还没跑（Composer 全空）→ 8 项都灰色「等待提示词」

**错误态**：

- 工作区路径不存在 → 「选其它工作区」按钮 + 跳 `/projects`
- 模型 token 过期 → 「修复 token」跳 `/settings?section=oauth`
- 发送失败（网络 / 后端 5xx）→ toast「发送失败：{message}（log_id: xxx）」+ Composer 内容保留

**边界**：

- 长按 Backspace 误删全文 → 不做 undo，但跳页提示「未保存改动，确定离开吗？」
- 附件拖拽落点不在 Composer 上 → 整个 `/new` 主区高亮接收
- 切工作区 → 重跑 Preflight 1（路径存在性）

### 1.3 `/tasks/:taskId` 任务详情（DetailRoute, 原型 03–06）

**用途**：任务运行中 / 已完成的对话流、产物、终端、日志。最重的页面。

**布局**：

```
[侧栏（会话列表，按 workspace 分组）] [主区三栏 or 两栏]

主区三栏：
  ┌──────────────┬──────────────────────────────┬──────────────┐
  │              │                              │              │
  │ 会话上下文栏  │ Chat Timeline                │ 右侧面板 Tab │
  │ - 任务摘要    │ - User / Assistant 轮次     │ - 文件 Diff  │
  │ - 工作区     │ - 工具调用块                  │ - 产物列表    │
  │ - 模型 / Skills│ - 思考块（可折叠）           │ - 终端流      │
  │ - 时间 / 成本 │ - 审批卡                     │ - 日志流      │
  │              │ - 自动滚到底                  │              │
  └──────────────┴──────────────────────────────┴──────────────┘
                  Composer（继续追问，sticky 底部）

⌘B 收起右侧面板 → 两栏布局；面板状态记 atom（不进 URL）
右侧 Tab 进 query：?panel=files|artifacts|terminal|logs（默认 files）
```

**Chat Timeline 元素**：

| 元素 | 渲染 | 交互 |
|------|------|------|
| 用户消息 | 右对齐气泡 | 长按复制 |
| 助手消息 | 左对齐 + markdown 渲染 | 代码块独立 + 复制按钮 |
| 工具调用 | 折叠卡 `read(path)` / `edit(path)` / `run(cmd)` | 点开看完整参数 + 结果 |
| 思考块 | 灰色折叠条「思考中...（X 秒）」 | 点开看完整 reasoning |
| 审批卡 | 黄色卡 + 「批准 / 拒绝」按钮 | 点按钮 → POST 后端 + 状态切换 |
| 错误 | 红色卡 + 错误描述 + `log_id` | 复制错误 / 跳 logs |

**主要数据**：

- 任务元信息（一次性拉，刷新页面用）
- SSE 事件流（实时）：消息增量、工具调用、思考、审批请求、产物生成
- 文件 Diff（按需拉）
- 产物清单（增量更新）
- 终端 / 日志（独立 SSE）

**空状态**：

- 任务刚创建、SSE 还没推第一个事件 → 显「Hermes 正在准备…」+ skeleton
- 任务已结束、无产物 → 产物 Tab 显「这个任务没有生成文件」

**错误态**：

| 场景 | 处理 |
|------|------|
| taskId 不存在 / 404 | 全屏「任务不存在或已删除」+ 「返回工作台」 |
| 无权访问（profile 切换后） | 全屏「这个任务属于另一个 profile，[切回 X profile]」 |
| SSE 连不上 | 顶部横条「连接断开，正在重连…」 |
| 工具调用执行失败 | 红色错误块在 Timeline 内 |

**关键交互**：

- 滚到底自动跟新（auto-scroll），用户手动上滚停止 auto-scroll；显「↓ 回到底部」按钮
- ⌘B 切右侧面板显隐
- 右侧切 Tab → URL `?panel=` 改
- Composer 续问 → 不跳页，新轮次追加到 Timeline 末尾
- 审批卡按钮 → 乐观更新（按钮 disable + 显「批准中」），失败回滚
- 长任务 → 浏览器标签页 favicon 显进度小点（v2 nice to have）

**边界**：

- Timeline 超过 1000 轮 → 虚拟化（仅渲染可视区 + 上下缓冲）
- 同时切多个任务 → 每个保留独立 SSE 连接，限 3 个并发，超过 LRU 关最早的
- 终端输出大量内容 → 行数限 5000，超过 deque pop 首
- 任务跑完后用户重新进入 → SSE replay 模式（拉完整历史）

### 1.4 `/history` 会话历史（HistoryRoute, 原型 19）

**用途**：全部会话表格 + 搜索 + 跳转。

**布局**：

```
[侧栏（项目快捷 + 进行中会话）]
[主区: 顶部搜索栏 + 过滤芯片 + 表格]

表格列：
  - 状态徽标（runing / done / failed / canceled）
  - 标题（点击 → /tasks/:id）
  - 模型
  - 工作区
  - 开始时间（相对 + tooltip 绝对）
  - 时长
  - 成本（token 数 / ￥ / $）
```

**字段**：

- 搜索框（query 参数 `?q=`）
- 过滤芯片：状态 / 模型 / 工作区 / 时间范围
- 排序：开始时间（默认降序）/ 时长 / 成本

**空状态 / 错误态**：

- 无任何会话 → 「还没有历史，[起一个新任务]」
- 搜索无结果 → 「未匹配到「{q}」的会话，[清除搜索]」
- API 失败 → 错误卡 + 重试

**关键交互**：

- 行点击 → `/tasks/:id`
- 行右键菜单（v2 nice to have）：「复制 ID」「删除」「重新跑」
- 分页：page-size 50，滚到底懒加载

**边界**：

- profile 切换 → 整表重新拉（不同 profile 的会话不互通）
- 任务还在跑时，表格行右侧显小转圈

## 2. 项目分组（02 顶导）

### 2.1 `/projects` 项目列表（ProjectsRoute, 原型 07）

**用途**：所有工作区（D1: 项目 = workspace UI 化）的总览。

**布局**：

```
[侧栏: 视图切换 + 跨项目进行中会话]
[主区: 网格 / 列表 (默认网格)]

每张项目卡：
  ├── 项目颜色（自动从路径 hash 出来或可改）
  ├── 项目名（workspace 路径最后一段，可重命名）
  ├── 完整路径（小字，tooltip 显示）
  ├── 会话计数（总数 / 进行中）
  ├── 最近活动时间
  └── Git 状态（分支 + 改动数）
```

**主要行动**：

- 卡片点击 → `/projects/:workspacePath`
- 右上角「+ 新项目」按钮 → 打开「添加工作区」modal（输入路径 / 选目录）
- 右键菜单：「重命名」「设主题色」「移除」（不删本地文件，仅从 Hermes 移除）

**空状态**：

- 「还没添加工作区，[选一个本地目录开始]」+ 按钮调原生文件夹对话框

**错误态**：

- 工作区路径不存在（用户外部删了）→ 项目卡变灰 + 「路径不存在 [移除]」按钮
- Git 状态获取失败 → 卡上 Git 区显「—」不显报错

### 2.2 `/projects/:workspacePath` 项目详情（ProjectDetailRoute, 原型 08）

**用途**：单个项目的概览、Git 状态、文件树、项目内会话。

**布局**：

```
[侧栏: 项目内子页导航 + 项目内进行中会话]
[主区从上到下]:
  ├── Hero（项目名 + 路径 + Git 分支 + 改动概要）
  ├── 项目操作按钮（在此项目下「新任务」/ 「打开终端」/ 「VSCode 打开」）
  ├── Git 状态卡（分支 / ahead-behind / 改动文件列表）
  ├── 文件树快照（前 N 条，超过显示「在 VSCode 中查看」）
  └── 项目内活动 Feed
```

**主要行动**：

- 「在此项目下新任务」→ `/new` + 工作区预填
- 「VSCode 打开」→ 调 OS 命令 `code <path>`（Tauri shell API）
- 「打开终端」→ macOS open Terminal / Windows 启 wt
- Git 改动文件点击 → 跳到 VSCode 对应文件（best-effort）

**空状态**：

- 路径存在但非 git 仓库 → Git 卡显「未初始化 Git」+ 「初始化」按钮（v2 nice to have，可推后）

**错误态**：

- 路径不存在 → 全页错误页 + 「回到项目列表」
- Git 命令超时 → Git 卡显「Git 状态未获取，[重试]」

## 3. 能力分组（03 顶导）

### 3.1 `/skills` Skills 列表（SkillsRoute, 原型 09）

**用途**：所有可用 Skills 的表格 + 启用/禁用 + Tab 过滤。

**布局**：

```
[侧栏: 子页 (Skills / MCP / Profiles*) + 来源过滤]
[主区: 搜索 + Tab 过滤 (全部/系统/我的/插件) + 表格]

表格列：
  - 启用开关（toggle）
  - 名称 + 描述（一行）
  - 触发器类型（prompt / file / cron / etc）
  - 来源（系统/我的/插件）
  - 使用次数
  - 操作（详情 / 编辑*）
```

\* Profiles 是否在能力侧栏由决策 9-A 决定。

**主要行动**：

- 启用开关 → 乐观更新，POST 后端
- 「我的」Tab 行 → 右侧出现「编辑」按钮 → 跳 `/skills/:skillId/edit`（决策 9-C）
- 「+ 新建 Skill」按钮 → 跳 `/skills/new/edit`

**空状态**：

- 全部 Tab 无数据 → 应该不存在（系统 Skills 总有）；若发生显「Skills 注册表为空，可能是后端启动异常」
- 「我的」Tab 无数据 → 「你还没写过 Skill，[从模板创建]」

**错误态**：

- 启用 / 禁用失败 → 开关回滚 + toast
- Skill 配置加载失败（编辑器进入时）→ 显错误页

### 3.2 `/skills/:skillId/edit` Skill 编辑器（待新增，原型 09b）

**用途**：编辑用户自定义 Skill 的 YAML / JSON 配置 + 测试运行。

**布局**：

```
[左: 「我的 Skills」列表（同 /skills?source=mine）]
[右: 编辑面板]
  ├── 元信息（name / description / triggers）
  ├── 代码编辑器（YAML 或 JSON，monaco-editor 风格）
  ├── 「测试运行」按钮 + 测试输入框
  ├── 「保存」「另存为」「删除」按钮
  └── 测试输出区
```

**主要行动**：

- 实时校验（输入时检查 schema）
- 保存：版本号自增，旧版本本地保留（v2 不入后端，便于 undo）
- 测试运行：调试模式，不写真实 IO
- 删除：二级确认

**错误态**：

- YAML / JSON 语法错 → inline 红色标记 + 阻止保存
- 测试运行失败 → 输出区显错误 + log_id

- 「重连」→ kill + respawn，期间状态显「启动中」+ 转圈

**空状态**：

**错误态**：

- 启动命令失败 → 服务器卡显 「启动失败」+ stderr 摘要 + 「查看完整日志」跳 /logs?source=mcp&q=:name

## 5. 可观测分组（05 顶导）

### 5.1 `/logs` 日志（LogsRoute, 原型 12）

**用途**：全局日志聚合 + 实时流 + 过滤搜索。

**布局**：

```
[侧栏: 子页 (日志 / 健康) + 来源过滤 (gateway / mcp / cron / skill / task)]
[主区]:
  ├── 过滤栏: level / source / 时间范围 / 关键字（全进 URL query, 决策 9-F 已定）
  ├── 实时流开关 (默认开)
  └── 日志表格 / 列表
       ├── 时间（相对 + 绝对 tooltip）
       ├── Level (DEBUG/INFO/WARN/ERROR)
       ├── Source
       ├── Message（截断 + 点击展开）
       └── 关联会话（若有，点击跳 /tasks/:id）
```

**URL 状态**：`?level=&source=&q=&since=&until=`

**主要行动**：

- 实时流开关：开 → SSE 跟新；关 → 拉静态结果，可后退翻页
- 单行点击 → 展开 JSON 详情
- 关键字搜索 → 防抖 300ms 后入 URL
- 「导出当前过滤结果」→ 下载 .jsonl 文件（v2 nice to have）

**空状态**：

- 过滤后无结果 → 「没有匹配的日志，[清除过滤]」
- 全部空 → 「Hermes 还没产生日志，可能是刚启动」

**错误态**：

- SSE 断 → 顶部横条
- 过滤超时（关键字太通用，结果太多）→ toast「结果过多，建议加时间范围」

**用途**：7 项系统健康探针 + 修复 hint。

